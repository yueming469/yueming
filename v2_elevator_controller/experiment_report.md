# 基于FPGA的电梯控制器设计实验报告

## 一、实验目的

1. 掌握使用Verilog HDL进行数字系统设计的方法
2. 理解有限状态机（FSM）的设计原理
3. 学习电梯控制器的设计方法
4. 完成FPGA设计的仿真验证
5. 掌握按键消抖、UART通信等外设驱动技术

## 二、实验要求

### 2.1 硬件平台
- **FPGA芯片**：Altera Cyclone II EP2C35F672C6（DE2实验板）
- **开发环境**：Quartus II 13.0+、ModelSim-Altera

### 2.2 功能要求
1. **楼层支持**：6层楼电梯控制
2. **按键控制**：8个独立按键
   - 6个楼层选择键（1-6楼）
   - 2个门控键（开门/关门）
3. **串口控制**：RS232串口通信，支持模式切换
   - 自动模式、手动模式、消防模式、检修模式
4. **显示输出**：12个LED指示灯
   - 6个楼层指示LED
   - 2个方向指示LED（上行/下行）
   - 4个模式指示LED（自动/手动/消防/检修）
5. **传感器**：6个拨码开关模拟楼层传感器
6. **蜂鸣器**：到达提示

## 三、系统设计

### 3.1 系统架构
本系统采用模块化分层架构，分为四层：
1. **底层驱动层**：按键消抖、UART收发、LED驱动、蜂鸣器驱动
2. **核心控制层**：状态机控制、呼梯调度、模式管理
3. **指令交互层**：按键解析、串口指令解析
4. **系统整合层**：顶层模块、信号同步

### 3.2 状态机设计
电梯控制器采用有限状态机设计，包含6个状态：
- **IDLE（空闲）**：电梯静止，等待请求
- **UP（上行）**：电梯向上运行
- **DOWN（下行）**：电梯向下运行
- **OPEN_DOOR（开门）**：电梯门打开
- **CLOSE_DOOR（关门）**：电梯门关闭
- **PROMPT（提示）**：到达提示（蜂鸣器响1秒）

### 3.3 运行模式
1. **自动模式**：根据呼梯请求自动调度
2. **手动模式**：通过门控按键手动控制开关门
3. **消防模式**：强制返回1楼并开门
4. **检修模式**：特殊维护模式

## 四、模块设计

### 4.1 顶层模块 (`Elevator_Top.v`)
连接所有子模块，定义系统接口：
- 输入：时钟、复位、8个按键、6个楼层传感器、UART接收
- 输出：6个楼层LED、2个方向LED、4个模式LED、蜂鸣器、门状态

### 4.2 按键消抖模块 (`Key_Debounce.v`)
- 对8个按键进行20ms消抖
- 采用三级寄存器消除亚稳态
- 输出高电平脉冲（仅按键释放时触发）

### 4.3 UART接收与模式解析模块 (`UART_Mode_Parse.v`)
- 实现9600bps串口接收
- 解析4字节ASCII指令（"AUTO"、"MANU"、"FIRE"、"CHEC"）
- 输出模式指令和更新标志

### 4.4 电梯核心控制模块 (`Elevator_Core.v`)
实现电梯的核心控制逻辑：
1. **运行模式更新逻辑**：响应串口指令切换模式
2. **当前楼层检测逻辑**：根据楼层传感器更新当前楼层
3. **呼梯信号登记逻辑**：登记按键呼梯请求
4. **目标楼层调度逻辑**：同向优先、就近响应算法
5. **状态机时序逻辑**：控制状态转换和延时
6. **输出逻辑**：控制LED、蜂鸣器、门状态

## 五、引脚分配

基于DE2实验板的引脚分配（详见`Elevator_Top.qsf`）：

| 信号 | 功能 | 引脚 |
|------|------|------|
| clk | 50MHz时钟 | PIN_12 |
| rst_n | 复位信号 | PIN_50 |
| key_raw[7:0] | 按键输入 | PIN_21-PIN_28 |
| uart_rx | UART接收 | PIN_29 |
| floor_sensor[5:0] | 楼层传感器 | PIN_43-PIN_48 |
| floor_led[5:0] | 楼层指示LED | PIN_31-PIN_36 |
| dir_led[1:0] | 方向指示LED | PIN_37-PIN_38 |
| mode_led[3:0] | 模式指示LED | PIN_39-PIN_42 |
| buzzer | 蜂鸣器 | PIN_49 |
| door_open | 门状态LED | PIN_25 |

## 六、仿真验证

### 6.1 测试平台设计
测试平台包含以下测试场景：
1. 按键控制电梯运行（自动模式）
2. 串口指令切换到消防模式
3. 楼层传感器变化模拟电梯运行
4. 手动模式开关门测试

### 6.2 仿真结果分析
通过仿真验证了设计的正确性：
- 按键消抖功能正常
- 状态机转换逻辑正确
- UART指令解析正确
- 各模式切换正常
- 蜂鸣器提示功能正常

## 七、实验总结

### 7.1 设计特点
1. **模块化设计**：系统结构清晰，易于维护和扩展
2. **状态机实现**：采用有限状态机，控制逻辑明确
3. **多种运行模式**：支持自动、手动、消防、检修四种模式
4. **外设驱动完善**：包含按键消抖、UART通信、LED控制等

### 7.2 技术要点
1. **按键消抖**：采用三级寄存器同步和计数器消抖
2. **UART通信**：实现9600bps串口接收和指令解析
3. **调度算法**：同向优先、就近响应的智能调度
4. **模式管理**：支持多种运行模式的切换和特殊处理

### 7.3 实验收获
1. 掌握了Verilog HDL的基本语法和设计方法
2. 理解了有限状态机在数字系统中的应用
3. 学习了电梯控制器的设计原理
4. 提高了模块化设计和系统集成能力
